= 安排報告
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
在您擁有想要排程定期產生和分發的檢視或 Excel 檔案後，您可以排程報表。

.開始之前
* 您必須具有應用程式管理員或儲存管理員角色。
* 您必須在 *常規* > *通知* 頁面中配置 SMTP 伺服器設置，以便報告引擎可以將報告作為電子郵件附件傳送至 Unified Manager 伺服器的收件者清單。
* 必須配置電子郵件伺服器以允許附件與產生的電子郵件一起傳送。


使用下列步驟測試並排程為視圖產生的報表。選擇或自訂您想要使用的視圖。以下步驟使用網路視圖來顯示網路介面的效能，但您可以使用任何您想要的視圖。

.步驟
. 開啟你的視野。此範例使用顯示 LIF 效能的預設網路視圖。在左側導覽窗格中，按一下「網路」>「網路介面」。
. 使用內建的 Unified Manager 功能根據需要自訂視圖。
. 自訂視圖後，您可以在「*視圖*」欄位中提供一個唯一的名稱，然後按一下複選標記進行儲存。
+
image::../media/view_save.gif[顯示如何儲存視圖的 UI 螢幕截圖。]

. 您可以使用 Microsoft® Excel 的進階功能來自訂您的報表。有關詳細信息，請參閱link:task_use_excel_to_customize_your_report.html["使用 Excel 自訂報告"]。
. 要在安排或共享之前查看輸出：
+
[cols="2*"]
|===
| 選項 | 描述 


 a| 
*如果您使用 Excel 自訂報表*
 a| 
查看現有的已下載的 Excel 檔案。



 a| 
*如果您沒有使用 Excel 自訂報表*
 a| 
將報告下載為 *CSV*、*PDF* 或 *XLSX* 檔案。

|===
+
使用已安裝的應用程式開啟文件，例如 Microsoft Excel (CSV/XSLX) 或 Adob​​e Acrobat (PDF)。

. 如果您對報告滿意，請點選「*計畫報告*」。
. 在報告計劃頁面中，按一下*新增計劃*。
. 接受預設名稱，即視圖名稱和頻率的組合，或自訂*計劃名稱*。
. 要首次測試預定報告，只需將自己新增為*收件者*。滿意後，新增所有報告收件者的電子郵件地址。
. 指定產生報告並傳送給收件人的頻率。您可以選擇*每日*、*每週*或*每月*。
. 選擇格式，*PDF*、*CSV* 或 *XSLX*。
+
[NOTE]
====
對於使用 Excel 自訂內容的報告，請始終選擇 *XSLX*。

====
. 按一下複選標記 (image:../media/blue_check.gif[""] ) 儲存報表計劃。
+
image::../media/scheduled_reports.gif[顯示如何儲存報告計畫的 UI 螢幕截圖。]

+
該報告將立即發送以進行測試。此後，將產生報告並按照預定頻率透過電子郵件發送給列出的收件者。
