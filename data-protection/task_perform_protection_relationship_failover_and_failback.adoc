= 執行保護關係故障轉移和故障回复
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
當保護關係中的來源磁碟區因硬體故障或災難而停用時，您可以使用 Unified Manager 中的保護關係功能使保護目標可讀/寫，並故障轉移到該磁碟區，直到來源再次聯機；然後，當原始來源可用於提供資料時，您可以故障還原至原始來源。

.開始之前
* 您必須具有應用程式管理員或儲存管理員角色。
* 您必須設定OnCommand Workflow Automation才能執行此操作。


.步驟
. link:task_break_snapmirror_relationship_from_health_volume_details.html["中斷SnapMirror關係"]。
+
您必須先中斷該關係，然後才能將目標從資料保護卷轉換為讀/寫卷，然後才能逆轉該關係。

. link:task_reverse_protection_relationships_from_health_volume_details.html["反轉保護關係"]。
+
當原始來源磁碟區再次可用時，您可能決定透過復原來源磁碟區來重新建立原始保護關係。在恢復來源之前，您必須將其與寫入先前目標的資料同步。您可以使用反向重新同步操作來建立新的保護關係，方法是反轉原始關係的角色並將來源磁碟區與先前的目標同步。為新的關係建立一個新的基線 Snapshot 副本。

+
反向關係看起來類似級聯關係：

+
image::../media/um_toplogy_reverse_resync.gif[顯示反向關係過程的 UI 螢幕截圖。]

. link:task_break_snapmirror_relationship_from_health_volume_details.html["中斷反向SnapMirror關係"]。
+
當原始來源磁碟區重新同步並可以再次提供資料時，使用中斷操作來中斷反轉關係。

. link:task_remove_protection_relationship_voldtls.html["刪除關係"]。
+
當不再需要逆轉的關係時，您應該先刪除該關係，然後再重新建立原始關係。

. link:task_resynchronize_protection_relationships_voldtls.html["重新同步關係"]。
+
使用重新同步操作將資料從來源同步到目標並重新建立原始關係。
