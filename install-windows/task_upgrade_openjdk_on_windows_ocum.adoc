= 升級 OpenJDK
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以在安裝了 Unified Manager 的 Windows 伺服器上升級到較新版本的 OpenJDK，以取得安全漏洞的修復。

.開始之前
您必須擁有安裝 Unified Manager 的系統的 Windows 管理員權限。

您可以在發行系列中更新 OpenJDK 版本。例如，您可以從 OpenJDK 11.0.16 升級到 OpenJDK 11.0.18，但不能直接從 OpenJDK 11 更新到 OpenJDK 12。

.步驟
. 以管理員使用者身分登入 Unified Manager 主機。
. 從 OpenJDK 網站下載適當版本的 OpenJDK（64 位元）到目標系統。
+
例如，下載 `jdk-11.0.18_windows-x64_bin.zip from https://www.oracle.com/in/java/technologies/javase/jdk11-archive-downloads.html`。

+

NOTE: 需要 Oracle 帳戶才能下載該檔案。如果您沒有 Oracle 帳戶，請造訪 link:https://signon.oracle.com/signin["Oracle 帳號登入"^] 頁面建立一個。

. 使用 Windows 服務控制台停止下列 Unified Manager 服務：
+
** NetApp Active IQ收購服務 (Ocie-au)
** NetApp Active IQ管理伺服器服務 (Oncommandsvc)


. 展開 `zip`文件。
. 從結果複製目錄和文件 `jdk`目錄（例如， `jdk-11.0.18`到安裝 Java 的位置。例子： `C:\Program Files\NetApp\JDK\`
. 使用 Windows 服務控制台啟動 Unified Manager 服務：
+
** NetApp Active IQ管理伺服器服務 (Oncommandsvc)
** NetApp Active IQ收購服務 (Ocie-au)
